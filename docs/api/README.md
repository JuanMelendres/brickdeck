# API Documentation

BrickDeck exposes a REST/JSON API from `apps/api`.

## Sources of Truth

| Artifact | Location | Notes |
| --- | --- | --- |
| Live OpenAPI spec | `GET /v3/api-docs` | Generated by springdoc — authoritative. |
| Swagger UI | `/swagger-ui/index.html` | Interactive, "try it out". |
| Static OpenAPI | [`openapi.yaml`](./openapi.yaml) | Hand-maintained summary for offline/portfolio use. |
| Endpoint reference | [`../architecture/api-design.md`](../architecture/api-design.md) | Human-readable contract + conventions. |
| Postman collection | [`postman/`](./postman/) | Auth, Catalog (incl. compare), Collection sets/parts. Import with the local environment. |

Local base URL: `http://localhost:8080`.

## Conventions

- Versioned under `/api/v1/*` (two legacy exceptions — see api-design.md).
- List endpoints return a `PageResponse<T>` envelope; detail endpoints return the DTO directly.
- Pagination: 0-indexed `page`, `size`, `sort` (e.g. `sort=id,asc`).
- Content type: `application/json`.

## Authentication

Collection endpoints require a JWT:

```
Authorization: Bearer <token>
```

Obtain a token via `POST /api/v1/auth/register` or `POST /api/v1/auth/login`
(returns `AuthResponse { token, tokenType, user }`). Catalog, auth, and health
endpoints are public. See [ADR-008](../decisions/ADR-008-jwt-stateless-auth.md).

## Error Codes

| Status | Meaning |
| --- | --- |
| 400 | Validation failure (`validationErrors` map in body). |
| 401 | Invalid credentials / missing or invalid token. |
| 404 | Resource missing or not owned by the caller. |
| 409 | Duplicate collection entry. |

## Examples

Register:

```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"me@example.com","password":"password123","displayName":"Me"}'
```

Search sets:

```bash
curl "http://localhost:8080/api/v1/sets/search?q=x-wing&page=0&size=10"
```

Add a set to the collection (authenticated):

```bash
curl -X POST http://localhost:8080/api/v1/collection/sets \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"setNumber":"75257-1","status":"OWNED","purchasePrice":159.99,"purchaseDate":"2026-01-15"}'
```
